Error → broken link

You seeLikely broken link
"No signing certificate found," "No account for team"Certificate — missing, expired, revoked, or not present in this Mac's Keychain
"No profiles for '<bundle id>' were found," "Doesn't include signing certificate"Provisioning profile — doesn't exist yet, or doesn't list the certificate you're building with
"Device is not registered," "not included in the profile"Registered device — the device isn't on the Development/Ad Hoc profile's device list
"App ID doesn't support <capability>," entitlement mismatchApp ID — the capability isn't checked on developer.apple.com, or you have a wildcard App ID where an explicit one is required
"Communication with Apple failed," provisioning profile expired mid-reviewNot a chain break — usually transient; retry, or check developer.apple.com's system status
Product → Archive greyed outNot signing at all — run destination is set to a Simulator (L0004); switch to a real device or "Any iOS Device"
fastest fix, most of the time With "Automatically manage signing" on, toggling it off and back on (or re-selecting your Team in Signing & Capabilities) forces Xcode to re-walk the whole chain and repair whichever link is stale. If that doesn't resolve it, the break is real — go to Certificates, Identifiers & Profiles directly and check the object the error names.
