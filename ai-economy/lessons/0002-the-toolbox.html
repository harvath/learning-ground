LESSON 0002 · THE TOOLBOX

MCP, agents, and the plumbing underneath

L0001 was who's building and who's paying. This lesson is what they're actually building with — the protocols and frameworks that turn a model into something that can act.

A raw model can only do one thing: turn text in, into text out. Everything that makes it feel like an "assistant" or "agent" — reading your files, calling an API, remembering yesterday's conversation — is a layer of tooling built on top. This lesson covers the load-bearing pieces of that layer: MCP (how models connect to tools and data), agent frameworks like Mastra (how you orchestrate multi-step agent behavior), ontology (a term currently being pulled in a new direction), and RAG (how models get access to facts they weren't trained on) — plus Hugging Face's actual current role in all of it.

MCP — the connector every agent now speaks

Model Context Protocol (MCP) is an open standard for connecting a model to external tools and data sources through one common interface, instead of every app writing its own bespoke integration for every tool. The common analogy is "USB-C for AI" — one physical connector, many devices.[1]

Anthropic created and announced MCP in November 2024. What matters for "is this current" purposes: in December 2025, Anthropic donated MCP to the Agentic AI Foundation, a new directed fund under the Linux Foundation, co-founded alongside Block and OpenAI — meaning MCP is no longer an Anthropic-only protocol, it's cross-vendor, neutrally governed infrastructure.[2] Adoption is real and large by late-2025 measures: 97M+ monthly SDK downloads and 10,000+ active MCP servers in production.[3] Of everything in this lesson, MCP is the term you can treat as settled, not in flux.

Agent frameworks — Mastra and the rest of the field

An agent framework is the code layer that orchestrates multi-step agent behavior on top of a model: deciding what tools to call and when, holding memory across steps, retrying failures, and logging/tracing what happened. Without one, "build an agent" means hand-rolling all of that yourself.

Mastra is the TypeScript-native entry in this field — agent orchestration, RAG pipelines, durable workflows, memory, and tracing, aimed at web developers already working in a Next.js/Vercel-style stack rather than the Python ML-research crowd.[4] It's built by the same three founders (Sam Bhagwat, Abhi Aiyer, Shane Thomas) who built Gatsby, went through Y Combinator, and reached its 1.0 release in January 2026.[5]

The overall shape as of 2026: after a fragmented 2025 (many competing frameworks), the field is consolidating around a handful of practical buckets — LangGraph for complex stateful workflows, vendor-native SDKs, and TypeScript-native options like Mastra for web teams.[6]

also worth knowing: AI-DLC AI-DLC ("AI-Driven Development Lifecycle") is AWS's own proposed methodology for software teams working alongside AI agents — small "AI Pods" instead of larger Scrum teams, short "bolts" instead of week-long sprints, with humans keeping approval gates over AI-drafted plans, code, and tests.[7] Unlike MCP, this is vendor-coined terminology, not an industry standard — useful to recognize when AWS or its partners use it, but don't assume it's a universally adopted framework name.

Ontology — a term currently being redefined

"Ontology" has two live meanings right now, and mixing them up is an easy tell that someone hasn't caught up with 2025–2026 usage. The classical sense, from knowledge representation and the semantic web, is a formal specification of the concepts, properties, and relationships in a domain (think RDF/OWL). The newer enterprise-AI sense, popularized specifically by Palantir, is a governed semantic layer sitting on top of an organization's raw data — every noun, relationship, rule, and permission modeled as a queryable, actionable object that AI agents act on directly instead of touching raw unstructured data.[8]

These aren't opposing definitions so much as the same root idea (a formal model of a domain) picking up a specific, branded, agent-oriented flavor in current enterprise discourse — Palantir's own agents are described as "natively ontology-aware," and the company has since exposed this layer to other frameworks via an "Ontology MCP" server, so the concept is starting to travel beyond Palantir's own product.[9] If you hear "ontology" in an AI-agent conversation in 2026, assume the Palantir-flavored sense unless context says otherwise.

RAG vs. long-context vs. fine-tuning

For about two years, the prediction was "context windows will keep growing until RAG (retrieval-augmented generation — fetching relevant documents and feeding them to the model at query time) becomes unnecessary." 2026 evidence says the opposite: a controlled comparison found brute-force long-context lost to RAG at scale — up to ~24x more expensive, and prone to losing facts before even hitting its advertised limit.[10]

The current working division of labor: fine-tuning changes how a model behaves (tone, format, task shape) — it's a poor tool for facts that change often. RAG supplies current, citable facts, and wins whenever the underlying data changes faster than the model gets retrained. Long-context is genuinely useful for one-off, whole-document reasoning — not as a RAG replacement at scale. Treat the three as complementary layers of one system, not competing techniques.

Hugging Face — no longer just a model zoo

Hugging Face is still the Hub where open models and datasets live, and the home of the transformers library — but by 2026 that undersells it. It now runs Inference Providers, a pay-as-you-go marketplace giving unified billing across 200+ models and multiple serving backends (integrated directly into GitHub Copilot Chat/VS Code), and it's pushed into open robotics after acquiring Pollen Robotics.[11] The right mental model in 2026: Hugging Face is open-model infrastructure and distribution — it brokers inference now, not just hosting.

Check it sticks

① What problem does MCP solve, and why is it now safe to treat as "settled" rather than "in flux"?

It gives models one common interface for connecting to external tools/data instead of a bespoke integration per tool. It's settled because Anthropic donated it to the Linux Foundation's Agentic AI Foundation (co-founded with Block and OpenAI) in Dec 2025 — it's now cross-vendor governed infrastructure with large real adoption, not a single company's proprietary protocol.

② Which layer, from L0001, does Hugging Face belong to — labs, infrastructure, capital, or none of those?

None of L0001's three cleanly — it's the toolbox layer (this lesson): open-model hosting, distribution, and now inference brokering, sitting on top of the labs/infra/capital structure rather than being a foundation-model lab or a chip/compute supplier itself.

③ You need a support bot to answer questions from a knowledge base that gets updated daily. Which approach fits best?

Fine-tuning is the worst fit for facts that change often — you'd be constantly retraining. Long-context works but loses on cost (up to ~24x more expensive at scale) and can drop facts before hitting its limit. RAG is built exactly for data that changes faster than a model gets retrained.
