LESSON 0003 · THE AGENT TOOLING FAMILY TREE

Protocols, agents, frameworks: who begat whom

L0002 introduced several tools in isolation. This lesson is the family tree — which category each one belongs to, and which ones are direct descendants of earlier ones.

The names in this space (goose, AGENTS.md, Semantic Kernel, LlamaIndex...) get thrown around as if they're all the same kind of thing. They aren't. Some are rules everyone agrees to follow. Some are software you install and run. Some are libraries you import to build your own agent. Sorting a name into the right bucket resolves most of the confusion before you even get to what it does.

Four kinds of thing

AGENTS.md — the README for agents

AGENTS.md is a plain markdown file at a repo's root — setup commands, code-style rules, test instructions, PR conventions — kept separate from the human-facing README so that stays concise. It exists because, before it, every agent vendor invented its own instructions file: .cursorrules for Cursor, CLAUDE.md for Claude Code, .github/copilot-instructions.md for Copilot — forcing teams to duplicate the same guidance for every tool or pick one to optimize for.[1]

It shipped in August 2025 as a genuine multi-vendor effort — OpenAI (Codex) as lead contributor, alongside Sourcegraph (Amp), Google (Jules), Cursor, and Factory — and is now used in 60,000+ open-source projects, read natively by Codex, Jules, Devin, VS Code, Cursor, Zed, Aider, Copilot, and more.[1]

a gotcha that affects how you work Claude Code (the tool you're reading this in) does not natively read AGENTS.md — it only reads CLAUDE.md. The common workaround is a one-line import directive inside CLAUDE.md pointing at AGENTS.md, so there's still one source of truth. Cursor, by contrast, reads AGENTS.md natively (including nested subdirectory files) as of 2026.[2]

goose — an agent you run, not a library you import

goose is Block's (formerly Square) open-source, terminal-first AI agent — you install and run it directly, and it reads/writes files, executes shell commands, and reaches external services via MCP, all driven by natural-language instructions. Built in Rust; 70+ MCP extensions; supports 15+ model providers including local Ollama.[3] The category it competes in is Claude Code and Aider — runnable agents — not Mastra or LangGraph, which you'd never "run" the same way; you build with them instead.

where these three meet On December 9, 2025, the Linux Foundation announced the Agentic AI Foundation (AAIF), founded with exactly three contributed projects: Anthropic's MCP, Block's goose, and OpenAI's AGENTS.md — one protocol, one runnable agent, one convention, now under neutral, cross-vendor governance. Platinum members include AWS, Anthropic, Block, Bloomberg, Cloudflare, Google, Microsoft, and OpenAI.[4] This is the concrete answer to "how do these relate": they don't just resemble each other, they were formally bundled together as the foundation's founding trio.

Two framework lineages that just merged

Two separate "old thing → new thing" mergers happened in this space within the last 18 months — worth knowing as mergers, not as two unrelated new names to memorize.

Swarm (Oct 2024) → OpenAI Agents SDK (Mar 2025)
Swarm was OpenAI's experimental, unmaintained agent-orchestration pattern. The Agents SDK is its production successor — minimal abstractions (Agents, Handoffs, Guardrails), provider-agnostic despite the OpenAI name.
Semantic Kernel (2023) + AutoGen → Microsoft Agent Framework (GA Apr 2026)
Semantic Kernel shipped in 2023 as enterprise "plumbing" for adding AI orchestration to existing .NET apps (state management, telemetry, connectors) — closer to "add AI to an app you already have" than "build a standalone autonomous agent." AutoGen was Microsoft's separate, research-driven multi-agent framework. On April 3, 2026 the two were unified into Microsoft Agent Framework, combining Semantic Kernel's enterprise plumbing with AutoGen's multi-agent orchestration patterns.[5] Both originals are now in maintenance mode (bug fixes only); Microsoft's own migration guide recommends moving off Semantic Kernel within 6–12 months.[5]

LlamaIndex — the retrieval specialist that didn't converge

LlamaIndex started life as GPT Index, open-sourced by Jerry Liu in November 2022 to connect GPT-3 to private data; it was renamed LlamaIndex in early 2023 to signal it wasn't OpenAI-specific (the timing lined up with Meta's Llama release).[6] Unlike Semantic Kernel and AutoGen, it did not converge into a general orchestration framework — its 2026 positioning is explicitly "more than a RAG framework, it is agentic document processing," doubling down on parsing, chunking, and retrieval rather than competing with LangGraph on orchestration.[7] The common production pattern: LlamaIndex handles retrieval, feeding into a separate agent framework (often LangGraph) that handles orchestration — used together, not as alternatives.

"Inference" — one word, three senses

You'll see "inference" used loosely across three genuinely different things. Worth disambiguating explicitly, since conflating them causes real confusion:

(a) The base ML sense — running a trained model to produce output, as opposed to training it. Every other sense below builds on this one.

(b) Hugging Face's Inference Providers — a marketplace/router giving pay-per-use access to 15+ hosted-inference partners (Groq, Together AI, Fireworks, Cerebras, and others) through one unified API. Hugging Face takes no markup on this — it's a funnel to keep developer attention in its ecosystem, not a direct profit center.[8]

(c) Inference-time / test-time compute — reasoning models spending extra computation at answer time (extended "thinking") to improve accuracy on hard problems, popularized by OpenAI's o1 (Sept 2024) and DeepSeek-R1 (Jan 2025). A reasoning-model query typically costs 30–100x the tokens of a normal chat query — and the technique's cost-efficiency depends heavily on the specific model, not just "spend more, get more": DeepSeek-R1 delivered comparable reasoning quality to o1 at roughly 27x cheaper API pricing at the time.[9]

Check it sticks

① goose and Mastra are both casually called "AI agent" tools. What's the concrete difference in how you'd actually use each one?

goose is a runnable agent — you install it and run it directly, like Claude Code or Aider. Mastra is a framework/SDK — you import it as a library to build your own custom agent application. One is software you run; the other is software you build with.

② Semantic Kernel and AutoGen used to be separate Microsoft projects. What are they now, and what should you do if you're maintaining code on either one?

They were merged into Microsoft Agent Framework (GA April 2026). Both originals are now in maintenance mode (bug fixes only, no new features) — Microsoft's own guidance recommends migrating off Semantic Kernel within 6–12 months.

③ A reasoning model spending extra tokens "thinking" before answering a hard question is an example of which sense of "inference"?

All reasoning-model "thinking" happens during inference in the base sense too, but the specific named phenomenon — spending extra compute at answer-time to trade latency/cost for accuracy — is inference-time (test-time) compute, popularized by o1 and DeepSeek-R1.

④ Does Claude Code (this tool) natively read AGENTS.md files as of 2026?

Claude Code reads CLAUDE.md only; there's an open, uncommitted GitHub issue requesting native AGENTS.md support. The workaround is a one-line import directive in CLAUDE.md pointing at AGENTS.md, so there's still a single source of truth. Cursor reads AGENTS.md natively.
