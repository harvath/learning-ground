Pipeline map

The whole system on one page — every lesson is a zoom-in on one node of this chain. Print-friendly.

the core idea Three systems that don't trust each other by default (your Mac, developer.apple.com, App Store Connect), bridged by two proof-files (certificate, provisioning profile). Every term in this course is "which system, which proof, which step."

Full chain

Xcode project
Code + .xcodeproj/project.pbxproj (L0003). Bundle ID set here.
optionally generated from project.yml via XcodeGen, instead of hand-maintained
Apple Developer Program membership → Team ID
Paid enrollment. Everything below is scoped to this Team ID (L0002).
App ID
Bundle ID registered on developer.apple.com + allowed capabilities. Nothing downstream can exist without this.
Certificate (Development / Distribution)
Proves who signed a build. Private key stays in your Mac's Keychain.
Provisioning profile
Bundles App ID + certificate(s) + entitlements + (dev/ad hoc only) device list.
Product → Archive
Release build, signed with the above, packaged as .xcarchive + .dSYMs (L0004).
Export / Distribute
Re-signed for a destination: App Store Connect, Ad Hoc, Enterprise, or Development.
App Store Connect
Receives the App Store Connect export. Processes it, surfaces it under TestFlight (L0005).
→ Internal testers: instant, no review
→ External testers: first build per version needs Beta App Review
Tester's device
Trusts the build because it trusts Apple's own re-signing — not a direct trust of you.
